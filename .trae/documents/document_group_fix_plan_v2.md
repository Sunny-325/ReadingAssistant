# 文档分组功能修复计划（版本2）

## 问题分析

1. **文档移动分组问题**：
   - 前端提示文档移动分组成功，但文档并没有移动到指定分组下面
   - 原因：`moveDocument` 函数可能没有实际实现移动逻辑

2. **分组标签点击问题**：
   - 点击分组标签最左侧可以展开该分组的列表
   - 点击整个分组标签没有反应，点击全部标签也没有反应
   - 原因：分组标签的点击事件没有正确绑定

3. **展开/折叠箭头问题**：
   - 分组标签左侧的展开/折叠箭头未实现
   - 原因：需要添加箭头图标和相应的点击事件

4. **返回列表问题**：
   - 点击返回列表按钮会返回到全部标签下的所有文档显示
   - 而不是返回到该分组下的所有文档的界面
   - 原因：返回列表的逻辑没有记住当前分组

## 修复方案

### 1. 修复文档移动分组功能

- **修改 `moveDocument` 函数**：
  - 实现实际的文档移动逻辑
  - 调用 `appStore.moveDocumentToGroup` 方法
  - 更新本地文档的 `group_id` 属性
  - 重新加载文档列表

### 2. 修复分组标签点击事件

- **修改分组标签的点击事件**：
  - 确保点击分组标签时，右侧区域显示该分组下的所有文档
  - 确保点击全部标签时，右侧区域显示所有文档
  - 修复 `selectedGroup` 的绑定逻辑

### 3. 实现展开/折叠箭头功能

- **添加展开/折叠箭头**：
  - 在分组标签左侧添加箭头图标
  - 点击箭头时展开/折叠分组下的文档列表
  - 确保箭头状态与分组展开状态同步

### 4. 修复返回列表功能

- **修改返回列表逻辑**：
  - 记住当前分组状态
  - 点击返回列表按钮时，返回到之前的分组视图
  - 而不是默认的全部视图
