# 中文文本阅读障碍辅助工具 - 技术文档

## 1. 架构设计

### 1.1 系统架构

#### 1.1.1 前端架构

- **框架**：Vue 3 + Composition API
- **状态管理**：Pinia
- **路由**：Vue Router
- **UI组件库**：Element Plus
- **构建工具**：Vite
- **HTTP客户端**：Axios

**核心组件**：
- `EditorView.vue`：文本编辑组件，支持文本输入、处理选项配置、示例文本加载
- `ReaderView.vue`：阅读组件，实现各种阅读模式和功能，包括双面板显示（处理后文字和简化文本）
- `SettingsView.vue`：设置面板，提供各种功能的配置选项
- `HistoryView.vue`：阅读历史组件，显示和管理阅读历史
- `DocumentView.vue`：文档管理组件，显示和管理用户文档
- `LoginView.vue`：登录组件
- `RegisterView.vue`：注册组件

**页面路由**：
- `/`：首页（重定向到 `/editor`）
- `/editor`：编辑器页面（默认首页）
- `/reader`：阅读器页面
- `/history`：阅读历史页面
- `/documents`：文档管理页面
- `/settings`：设置页面
- `/login`：登录页面
- `/register`：注册页面

#### 1.1.2 后端架构

- **框架**：FastAPI
- **数据库**：MySQL 8.0
- **缓存**：Redis 7.0+
- **ORM**：SQLAlchemy
- **模型**：通过阿里云通义千问API调用Qwen大语言模型
- **认证**：JWT

**核心服务**：
- `text_processor.py`：文本处理服务，实现12项核心阅读辅助功能
- `qwen_model_manager.py`：Qwen模型管理器，通过API调用大语言模型进行语义分析
- `task_manager.py`：任务管理器，支持异步任务处理和断点续传
- `file_service.py`：文件服务，实现文件上传、解析和导出
- `voice_service.py`：语音服务，使用pyttsx3实现文本转语音
- `auth_service.py`：认证服务，实现用户认证和授权，支持用户注销功能

#### 1.1.3 数据流程

1. 用户在前端输入或上传文本
2. 在编辑器中选择处理选项（意群划分、文本简化、主次内容区分）
3. 前端将文本发送到后端API `/api/text/process`
4. 后端调用文本处理服务进行初步处理（分词、词性标注）
5. 根据需要调用Qwen模型进行语义分析（意群划分、文本简化、主次内容区分）
6. 将处理结果返回给前端，包含原始文本、处理后文本、简化文本、意群划分、词性标注等信息
7. 前端展示处理结果，提供双面板阅读、语音朗读、词语释义查询等功能

## 3. 算法说明

### 3.1 意群划分算法

**算法原理**：
- 基于规则和模型的混合方法
- 对于短文本（<1000字），使用基于标点和词语数的规则划分
- 对于长文本，使用Qwen大语言模型进行语义分析

**实现步骤**：
1. 检查文本长度，选择合适的划分方法
2. 短文本：按句子分割，然后按词语数（默认4个）划分意群
3. 长文本：使用模型生成语义相关的意群
4. 为每个意群添加ID、文本内容、起始位置、结束位置和重要性评分
5. 处理划分失败的情况，使用降级方案

**参数设置**：
- 意群大小：轻度划分约6-7个词，中度划分约3-4个词，高度划分约1-3个词
- 短文本阈值：1000字
- 长文本分块大小：1500字

### 3.2 文本简化算法

**算法原理**：
- 基于规则和模型的混合方法
- 使用规则替换复杂词汇和句式
- 使用模型生成更简单的表达方式
- 支持三级简化程度：轻度、中度、深度

**实现步骤**：
1. 检查文本长度，选择合适的简化方法
2. 短文本：使用规则替换复杂词汇，拆分长句子
3. 长文本：使用模型生成简化后的文本
4. 对模型简化结果进行清理和验证
5. 如果简化失败，使用基于规则的降级方案

**参数设置**：
- 简化程度：轻度(1)、中度(2)、深度(3)
- 短文本阈值：1000字
- 模型生成温度：0.2
- 最大生成长度：8192

### 3.3 主次内容区分算法

**算法原理**：
- 基于规则和模型的混合方法
- 短文本使用基于位置的规则
- 长文本使用模型分析内容重要性

**实现步骤**：
1. 检查文本长度，选择合适的区分方法
2. 短文本：将第一个和最后一个意群标记为主要内容
3. 长文本：使用模型分析每个意群的重要性（0-1）
4. 使用混合方法确定主次内容：
   - 绝对阈值：低于0.5的为次要内容
   - 强制最低比例：确保至少30%的内容被标记为次要内容
5. 为每个意群添加is_primary标记

**参数设置**：
- 短文本阈值：1000字
- 重要性阈值：0.5
- 次要内容最低比例：30%

### 3.4 词性标注算法

**算法原理**：
- 使用jieba进行中文分词和词性标注
- 用户可选择要标注的词性（默认选中名词、动词、形容词）
- 为选中的词性分配不同颜色和字号
- 未选中的词性不显示特殊标注

**词性类型**：

| 词性代码 | 词性名称 | 默认颜色 |

### 3.5 实时语音高亮算法

**算法原理**：
- 基于文本长度和语音播放进度的映射
- 实时计算当前朗读的词语位置

**实现步骤**：
1. 将文本分割为词语数组
2. 开始语音播放，记录当前播放时间
3. 根据播放进度计算当前朗读的词语索引
4. 更新前端高亮显示
5. 实现滚动同步，确保当前朗读的词语在可视区域

**技术要点**：
- 使用pyttsx3进行语音合成
- 实现单词级别的精确高亮
- 支持双面板独立控制

## 4. API文档

### 4.1 认证相关API

#### POST /api/auth/token
- **功能**：用户登录
- **请求体**：form-data格式
  ```
  username: 用户名
  password: 密码
  ```
- **响应**：
  ```json
  {
    "access_token": "JWT令牌",
    "token_type": "bearer"
  }
  ```

#### POST /api/auth/register
- **功能**：用户注册
- **请求体**：
  ```json
  {
    "username": "用户名",
    "email": "邮箱",
    "password": "密码"
  }
  ```
- **响应**：用户信息对象

#### GET /api/auth/me
- **功能**：获取当前用户信息
- **请求头**：Authorization: Bearer {token}
- **响应**：用户信息对象

#### POST /api/auth/logout
- **功能**：退出登录，结束当前会话（token仍然有效直到过期）
- **请求头**：Authorization: Bearer {token}
- **响应**：
  ```json
  {
    "message": "退出登录成功"
  }
  ```

#### PUT /api/auth/update-username
- **功能**：修改用户名
- **请求头**：Authorization: Bearer {token}
- **请求体**：
  ```json
  {
    "new_username": "新用户名"
  }
  ```
- **响应**：
  ```json
  {
    "message": "用户名修改成功"
  }
  ```

#### PUT /api/auth/update-password
- **功能**：修改密码
- **请求头**：Authorization: Bearer {token}
- **请求体**：
  ```json
  {
    "old_password": "旧密码",

  ```
- **响应**：

  {
    "message": "密码修改成功"

### 5.2 系统优化

#### 5.2.1 Redis 分布式缓存

##### 5.2.1.1 用途概述

Redis 在系统中主要用于以下场景：

| 用途 | 说明 | 实现位置 |
|------|------|----------|
| **任务状态管理** | 存储异步任务的状态和进度信息，提供快速查询 | `core/task_manager.py` |
| **分布式限流** | 实现用户任务创建速率限制，防止系统过载 | `core/task_manager.py` |
| **优雅降级** | Redis 不可用时自动降级到本地缓存，保证系统可用性 | `core/redis_manager.py` |

##### 5.2.1.2 任务状态管理

**实现机制**：
- 任务创建时，将状态写入 Redis（键格式：`task:{task_id}:status`）
- 任务状态更新时（处理中、暂停、完成、失败），同步更新 Redis
- **查询任务进度时，优先从 Redis 获取，提高响应速度**
- **Redis 不可用时自动降级到数据库查询**
- 任务完成后自动清理 Redis 中的状态（过期时间：1小时）

**数据结构**：
```json
{
  "status": "processing",
  "updated_at": "2024-01-15T10:30:00",
  "progress": {
    "group_index": 0,
    "total_groups": 5,
    "completed_chunks": 2,
    "total_chunks": 4
  }
}
```

**核心优势**：
- ✅ **高性能查询**：任务状态查询优先从 Redis 获取，避免频繁数据库访问
- ✅ **优雅降级**：Redis 不可用时自动降级到数据库查询
- ✅ **实时更新**：支持实时进度更新，提高响应速度

##### 5.2.1.3 分布式限流

**实现机制**：
- 使用滑动窗口算法实现速率限制
- 键格式：`rate_limit:{user_id}`
- 配置：每分钟最多创建 5 个任务

**限流逻辑**：
```
1. 用户请求创建任务时，检查 Redis 中的计数器
2. 如果计数器 >= 限流阈值，拒绝请求（HTTP 429）
3. 如果计数器 < 限流阈值，增加计数器并允许请求
4. 计数器设置自动过期（60秒），实现滑动窗口
```

##### 5.2.1.4 配置与依赖

**依赖**：
- `redis==5.0.0`

**配置项**（`settings.py`）：
```python
REDIS_HOST = "localhost"
REDIS_PORT = 6379
REDIS_DB = 0
```

- 实时语音高亮：实现单词级别的精确高亮同步
- 用户友好的词语释义查询：支持用户主动查询词语含义
- 词性颜色和字号区分：不同词性使用不同颜色和字号显示
- 可选词性标注：用户可选择要标注的词性，默认标注名词、动词、形容词
- 语音合成：使用pyttsx3实现文本转语音，无需网络连接

未来，系统将继续优化性能，扩展功能，提升智能化水平，为更多阅读障碍患者提供帮助。
