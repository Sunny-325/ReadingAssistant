# 中文文本阅读障碍辅助工具 - 技术文档

## 架构设计

### 1. 系统架构

#### 1.1 前端架构

- **框架**：Vue 3 + Composition API
- **状态管理**：Pinia
- **路由**：Vue Router
- **UI组件库**：Element Plus
- **构建工具**：Vite
- **HTTP客户端**：Axios

**核心组件**：
- `EditorView.vue`：文本编辑组件，支持文本输入、处理选项配置、示例文本加载
- `ReaderView.vue`：阅读组件，实现各种阅读模式和功能，包括双面板显示（处理后文字和简化文本）
- `SettingsView.vue`：设置面板，提供各种功能的配置选项
- `HistoryView.vue`：阅读历史组件，显示和管理阅读历史
- `LoginView.vue`：登录组件
- `RegisterView.vue`：注册组件

**页面路由**：
- `/editor`：编辑器页面（默认首页）
- `/reader`：阅读器页面
- `/history`：阅读历史页面
- `/settings`：设置页面
- `/login`：登录页面
- `/register`：注册页面

#### 1.2 后端架构

- **框架**：FastAPI
- **数据库**：MySQL 8.0
- **ORM**：SQLAlchemy
- **模型**：通过阿里云通义千问API调用Qwen大语言模型
- **认证**：JWT

**核心服务**：
- `text_processor.py`：文本处理服务，实现12项核心阅读辅助功能
- `qwen_model_manager.py`：Qwen模型管理器，通过API调用大语言模型进行语义分析
- `file_service.py`：文件服务，实现文件上传、解析和导出
- `voice_service.py`：语音服务，使用pyttsx3实现文本转语音

#### 1.3 数据流程

1. 用户在前端输入或上传文本
2. 在编辑器中选择处理选项（意群划分、文本简化、主次内容区分）
3. 前端将文本发送到后端API `/api/text/process`
4. 后端调用文本处理服务进行初步处理（分词、词性标注）
5. 根据需要调用Qwen模型进行语义分析（意群划分、文本简化、主次内容区分）
6. 将处理结果返回给前端，包含原始文本、处理后文本、简化文本、意群划分、词性标注等信息
7. 前端展示处理结果，提供双面板阅读、语音朗读、词语释义查询等功能

## 算法说明

### 1. 意群划分算法

**算法原理**：
- 基于规则和模型的混合方法
- 对于短文本（<1000字），使用基于标点和词语数的规则划分
- 对于长文本，使用Qwen大语言模型进行语义分析

**实现步骤**：
1. 检查文本长度，选择合适的划分方法
2. 短文本：按句子分割，然后按词语数（默认4个）划分意群
3. 长文本：使用模型生成语义相关的意群
4. 为每个意群添加ID、文本内容、起始位置、结束位置和重要性评分
5. 处理划分失败的情况，使用降级方案

**参数设置**：
- 意群大小：轻度划分约6-7个词，中度划分约3-4个词，高度划分约1-3个词
- 短文本阈值：1000字
- 长文本分块大小：1200字

### 2. 文本简化算法

**算法原理**：
- 基于规则和模型的混合方法
- 使用规则替换复杂词汇和句式
- 使用模型生成更简单的表达方式
- 支持三级简化程度：轻度、中度、深度

**实现步骤**：
1. 检查文本长度，选择合适的简化方法
2. 短文本：使用规则替换复杂词汇，拆分长句子
3. 长文本：使用模型生成简化后的文本
4. 对模型简化结果进行清理和验证
5. 如果简化失败，使用基于规则的降级方案

**参数设置**：
- 简化程度：轻度(1)、中度(2)、深度(3)
- 短文本阈值：1000字
- 模型生成温度：0.3
- 最大生成长度：4096

### 3. 主次内容区分算法

**算法原理**：
- 基于规则和模型的混合方法
- 短文本使用基于位置的规则
- 长文本使用模型分析内容重要性

**实现步骤**：
1. 检查文本长度，选择合适的区分方法
2. 短文本：将第一个和最后一个意群标记为主要内容
3. 长文本：使用模型分析每个意群的重要性（0-1）
4. 使用混合方法确定主次内容：
   - 绝对阈值：低于0.5的为次要内容
   - 强制最低比例：确保至少30%的内容被标记为次要内容
5. 为每个意群添加is_primary标记

**参数设置**：
- 短文本阈值：1000字
- 重要性阈值：0.5
- 次要内容最低比例：30%

### 4. 词性标注算法

**算法原理**：
- 使用jieba进行中文分词和词性标注
- 用户可选择要标注的词性（默认选中名词、动词、形容词）
- 为选中的词性分配不同颜色和字号
- 未选中的词性不显示特殊标注

**词性选择**：
- 用户可在设置页面选择要标注的词性
- 默认选中：名词(n)、动词(v)、形容词(a)
- 可选词性包括：名词、动词、形容词、副词、介词、连词、助词、代词、数词、量词、时间词、处所词、方位词、区别词、状态词、叹词、语气词、拟声词

### 5. 实时语音高亮算法

**算法原理**：
- 基于文本长度和语音播放进度的映射
- 实时计算当前朗读的词语位置

**实现步骤**：
1. 将文本分割为词语数组
2. 开始语音播放，记录当前播放时间
3. 根据播放进度计算当前朗读的词语索引
4. 更新前端高亮显示
5. 实现滚动同步，确保当前朗读的词语在可视区域

**技术要点**：
- 使用pyttsx3进行语音合成
- 实现单词级别的精确高亮
- 支持双面板独立控制

**功能说明**：
- 用户可在设置页面自定义语音参数
- 使用pyttsx3进行文本转语音
- 支持语音速度、音量的调节
- 设置实时生效，无需重新启动

**TTS实现方式**：
- **pyttsx3**：
  - 使用后端pyttsx3库进行文本转语音
  - 通过API调用后端服务生成语音
  - 支持语音速度、音量调节
  - 返回音频数据，使用HTML5 Audio播放
  - 不需要网络连接，响应速度快
  - 完全免费，无调用次数限制

## 总结

中文文本阅读障碍辅助工具是一个基于AI技术的阅读辅助系统，旨在帮助阅读障碍患者提高阅读能力。系统采用了前后端分离的架构，通过阿里云通义千问API集成了Qwen大语言模型，实现了12项核心功能。

系统的设计考虑了性能优化、安全性和扩展性，能够在普通CPU设备上流畅运行。系统的架构和算法设计确保了系统的可靠性和高效性，能够满足不同用户的需求。

核心技术特点：
- 混合处理策略：对短文本使用基于规则的快速处理，对长文本使用AI模型分析
- 双面板设计：同时显示处理后文字和简化文本，方便用户对比
- 编辑器集成处理选项：用户可在编辑器中直接选择意群划分、文本简化程度、主次内容区分
- 缓存机制：实现多层缓存，提高处理速度
- 长文本分块：通过分块处理支持长文本的高效处理
- 实时语音高亮：实现单词级别的精确高亮同步
- 用户友好的词语释义查询：支持用户主动查询词语含义
- 词性颜色和字号区分：不同词性使用不同颜色和字号显示
- 可选词性标注：用户可选择要标注的词性，默认标注名词、动词、形容词
- 语音合成：使用pyttsx3实现文本转语音，无需网络连接

未来，系统将继续优化性能，扩展功能，提升智能化水平，为更多阅读障碍患者提供帮助。
