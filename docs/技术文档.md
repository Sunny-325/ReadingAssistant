# 中文文本阅读障碍辅助工具 - 技术文档

## 1. 架构设计

### 1.1 系统架构

#### 1.1.1 前端架构

- **框架**：Vue 3 + Composition API
- **状态管理**：Pinia
- **路由**：Vue Router
- **UI组件库**：Element Plus
- **构建工具**：Vite
- **HTTP客户端**：Axios

**核心组件**：
- `EditorView.vue`：文本编辑组件，支持文本输入、处理选项配置、示例文本加载
- `ReaderView.vue`：阅读组件，实现各种阅读模式和功能，包括双面板显示（处理后文字和简化文本）
- `SettingsView.vue`：设置面板，提供各种功能的配置选项
- `HistoryView.vue`：阅读历史组件，显示和管理阅读历史
- `DocumentView.vue`：文档管理组件，显示和管理用户文档
- `LoginView.vue`：登录组件
- `RegisterView.vue`：注册组件

**页面路由**：
- `/`：首页（重定向到 `/editor`）
- `/editor`：编辑器页面（默认首页）
- `/reader`：阅读器页面
- `/history`：阅读历史页面
- `/documents`：文档管理页面
- `/settings`：设置页面
- `/login`：登录页面
- `/register`：注册页面

#### 1.1.2 后端架构

- **框架**：FastAPI
- **数据库**：MySQL 8.0
- **缓存**：Redis 7.0+
- **ORM**：SQLAlchemy
- **模型**：通过阿里云通义千问API调用Qwen大语言模型
- **认证**：JWT

**核心服务**：
- `text_processor.py`：文本处理服务，实现12项核心阅读辅助功能
- `qwen_model_manager.py`：Qwen模型管理器，通过API调用大语言模型进行语义分析
- `task_manager.py`：任务管理器，支持异步任务处理和断点续传
- `file_service.py`：文件服务，实现文件上传、解析和导出
- `voice_service.py`：语音服务，使用pyttsx3实现文本转语音（离线）
- `auth_service.py`：认证服务，实现用户认证和授权，支持用户注销功能

#### 1.1.3 数据流程

1. 用户在前端输入或上传文本
2. 在编辑器中选择处理选项（意群划分、文本简化、主次内容区分）
3. 前端将文本发送到后端API `/api/text/process`
4. 后端调用文本处理服务进行初步处理（分词、词性标注）
5. 根据需要调用Qwen模型进行语义分析（意群划分、文本简化、主次内容区分）
6. 将处理结果返回给前端，包含原始文本、处理后文本、简化文本、意群划分、词性标注等信息
7. 前端展示处理结果，提供双面板阅读、语音朗读、词语释义查询等功能

## 3. 算法说明

### 3.1 意群划分算法

**算法原理**：
- 基于规则和模型的混合方法
- 对于短文本（<1000字），使用基于标点和词语数的规则划分
- 对于长文本，使用Qwen大语言模型进行语义分析

**实现步骤**：
1. 检查文本长度，选择合适的划分方法
2. 短文本：按句子分割，然后按词语数（默认4个）划分意群
3. 长文本：使用模型生成语义相关的意群
4. 为每个意群添加ID、文本内容、起始位置、结束位置和重要性评分
5. 处理划分失败的情况，使用降级方案

**参数设置**：
- 意群大小：轻度划分约6-7个词，中度划分约3-4个词，高度划分约1-3个词
- 短文本阈值：1000字
- 长文本分块大小：1500字

### 3.2 文本简化算法

**算法原理**：
- 基于规则和模型的混合方法
- **三层回退机制**：LLM简化 → 重试简化 → 基于规则简化 → 返回原文
- 使用规则替换复杂词汇和句式
- 使用模型生成更简单的表达方式
- 支持三级简化程度：轻度、中度、深度

**三层回退流程**：
```
第一层：LLM智能简化（Primary）
  ↓ (失败或异常)
第二层：重试简化（Retry）
  ↓ (失败或异常)
第三层：基于规则的简化（Fallback）
  ↓ (全部失败)
返回原始文本
```

**基于规则的简化方法**：
1. **词汇替换**：维护70+组的复杂词→简单词映射表
   - "此外" → "另外"
   - "因此" → "所以"
   - "然而" → "但是"
   - 等等...

2. **冗余词移除**：直接删除过度修饰的副词
   - "非常"、"极其"、"十分"、"特别"等

3. **长句拆分**：使用正则表达式在特定连接词处拆分
   - "...，此外..." → "...。此外..."
   - "...，因此..." → "...。因此..."

4. **级别差异化处理**：
   - 轻度：仅词汇替换和冗余词移除
   - 中度：增加长句拆分
   - 深度：删除括号内容、例子等

**参数设置**：
- 简化程度：轻度(1)、中度(2)、深度(3)
- 短文本阈值：1000字
- 模型生成温度：0.2
- 最大生成长度：8192

### 3.3 主次内容区分算法

**算法原理**：
- 基于规则和模型的混合方法
- 短文本使用基于位置的规则
- 长文本使用模型分析内容重要性

**实现步骤**：
1. 检查文本长度，选择合适的区分方法
2. 短文本：将第一个和最后一个意群标记为主要内容
3. 长文本：使用模型分析每个意群的重要性（0-1）
4. 使用混合方法确定主次内容：
   - 绝对阈值：低于0.5的为次要内容
   - 强制最低比例：确保至少30%的内容被标记为次要内容
5. 为每个意群添加is_primary标记

**参数设置**：
- 短文本阈值：1000字
- 重要性阈值：0.5
- 次要内容最低比例：30%

### 3.4 词性标注算法

**算法原理**：
- 使用jieba进行中文分词和词性标注
- 用户可选择要标注的词性（默认选中名词、动词、形容词）
- 为选中的词性分配不同颜色和字号
- 未选中的词性不显示特殊标注

**词性映射表**：

| 词性代码 | 词性名称 | 默认颜色 |
|----------|----------|----------|
| n | 名词 | #E74C3C (红色) |
| v | 动词 | #3498DB (蓝色) |
| a | 形容词 | #2ECC71 (绿色) |
| d | 副词 | #F39C12 (橙色) |
| p | 介词 | #9B59B6 (紫色) |
| c | 连词 | #1ABC9C (青色) |
| u | 助词 | #95A5A6 (灰色) |
| r | 代词 | #E67E22 (深橙色) |
| m | 数词 | #00CED1 (暗青) |
| q | 量词 | #9370DB (中紫) |
| t | 时间词 | #FF6347 (番茄红) |

### 3.5 语音朗读同步算法

**算法原理**：
- 音频时间进度 → 字符位置 → 意群映射 → 高亮更新
- 使用 requestAnimationFrame 实现 60fps 流畅同步

**实现步骤**：
1. **构建意群位置映射表**：计算每个意群在全文中的字符起始/结束位置
2. **计算朗读进度**：progress = currentTime / totalDuration
3. **计算当前字符位置**：currentCharOffset = progress × totalChars
4. **查找对应意群**：遍历映射表，找到当前字符所属的意群
5. **更新高亮状态**：currentOriginalSegment = segmentId
6. **自动滚动**：scrollToElement() 将意群滚动到蒙版可见区域中央

**蒙版居中计算**：
```javascript
visibleHeight = lineHeight × maskLines × fontSize
scrollPosition = targetOffsetTop - visibleHeight / 2
```

### 3.6 词语释义上下文提取算法

**算法原理**：
- 基于句子边界的上下文精准获取
- 保证语义完整性

**实现步骤**：
1. 使用 `indexOf()` 精确定位词语位置
2. 向前查找最近的句子边界（句号、感叹号、问号等）
3. 向后查找最近的句子边界
4. 在边界处截取，确保词语不被截断
5. 如果上下文太短，自动扩展到最小长度
6. 用 `...` 标记省略部分

## 4. 核心API接口

### 4.1 文本处理接口

**POST /api/text/process**

请求参数：
```json
{
  "text": "原始文本内容",
  "options": {
    "enable_segmentation": true,
    "segment_level": 2,
    "enable_simplification": true,
    "simplify_level": 1,
    "enable_primary_secondary": true,
    "enable_pos_tagging": true
  }
}
```

响应：
```json
{
  "success": true,
  "data": {
    "original_text": "原始文本",
    "segments": [...],
    "simplified_text": "简化文本",
    "simplified_segments": [...],
    "pos_tags": [...],
    "primary_content": [...],
    "secondary_content": [...]
  }
}
```

### 4.2 词语释义接口

**GET /api/text/definition/{word}?context=...**

响应：
```json
{
  "word": "公园",
  "phonetic": "gōng yuán",
  "definitions": ["供公众游览休息的园林"],
  "examples": ["周末我们去公园野餐。"],
  "contextual_meaning": "指作者要去散步休闲的场所"
}
```

### 4.3 语音合成接口

**POST /api/voice/tts**

请求参数：
```json
{
  "text": "要转换的文本",
  "rate": 150,
  "volume": 1.0
}
```

响应：WAV音频数据
