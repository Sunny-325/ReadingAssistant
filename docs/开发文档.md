# 中文文本阅读障碍辅助工具 - 开发文档

## API文档

### 1. 文本处理相关API

#### POST /api/text/process
- **功能**：处理文本，应用各种阅读辅助功能（适用于短文本）
- **请求体**：
  ```json
  {
    "text": "要处理的文本",
    "options": {
      "enableChunk": true,  // 启用意群划分
      "chunkLevel": 2,  // 意群划分程度：1=轻度, 2=中度, 3=高度
      "enableMainContent": true,  // 启用主次内容区分
      "enableSimplify": true,  // 启用文本简化
      "simplifyLevel": 1,  // 简化程度：1=轻度, 2=中度, 3=深度
      "pos_tagging": true  // 启用词性标注
    },
    "document_id": null  // 可选，文档ID
  }
  ```
- **响应**：
  ```json
  {
    "message": "文本处理成功",
    "data": {
      "original_text": "原始文本",
      "processed_text": "处理后的文本",
      "simplifiedContent": "简化后的文本",
      "segments": [
        {
          "id": 1,
          "text": "意群1",
          "start_pos": 0,
          "end_pos": 10,
          "importance": 0.8,
          "is_primary": true
        }
      ],
      "simplified_segments": [],
      "pos_tags": [],
      "simplified_pos_tags": [],
      "metadata": {
        "is_long_text": false,
        "chunk_count": 0,
        "total_processing_time": 0.5,
        "processing_time_ms": 500,
        "text_length": 100
      }
    },
    "document_id": 1
  }
  ```

#### POST /api/text/process/async
- **功能**：异步处理文本，应用各种阅读辅助功能（适用于长文本）
- **请求体**：
  ```json
  {
    "text": "要处理的文本",
    "options": {},
    "document_id": null
  }
  ```
- **响应**：
  ```json
  {
    "message": "任务创建成功",
    "data": {
      "task_id": 1,
      "document_id": 1
    }
  }
  ```

#### GET /api/text/process/async/{task_id}
- **功能**：获取异步任务状态
- **路径参数**：
  - `task_id`：任务ID
- **响应**：
  ```json
  {
    "message": "获取任务状态成功",
    "data": {
      "task_id": 1,
      "status": "completed",  // pending/processing/completed/failed
      "created_at": "2023-01-01T12:00:00",
      "result": {},  // 任务完成时返回
      "error": "错误信息"  // 任务失败时返回
    }
  }
  ```

#### GET /api/text/definition/{word}
- **功能**：获取词语释义
- **路径参数**：
  - `word`：要查询的词语
- **查询参数**：
  - `context`：可选，词语的上下文
- **响应**：
  ```json
  {
    "message": "获取词语释义成功",
    "data": {
      "word": "查询的词语",
      "phonetic": "读音",
      "definitions": ["释义1", "释义2"],
      "examples": ["例句1", "例句2"],
      "raw_response": "详细说明"
    }
  }
  ```

#### POST /api/text/simplify
- **功能**：简化文本
- **请求体**：
  ```json
  {
    "text": "要简化的文本",
    "level": 1  // 简化程度：1=轻度, 2=中度, 3=深度
  }
  ```
- **响应**：
  ```json
  {
    "message": "文本简化成功",
    "data": {
      "original_text": "原始文本",
      "simplifiedContent": "简化后的文本"
    }
  }
  ```

#### POST /api/text/chunk
- **功能**：意群划分
- **请求体**：
  ```json
  {
    "text": "要划分的文本",
    "chunk_size": 5  // 分块大小（1-20）
  }
  ```
- **响应**：
  ```json
  {
    "message": "意群划分成功",
    "data": {
      "text": "原始文本",
      "segments": [
        {
          "id": 1,
          "text": "意群1",
          "start_pos": 0,
          "end_pos": 10,
          "importance": 1.0
        }
      ]
    }
  }
  ```

  ```json
      "id": 1,
      "title": "文档标题",
      "group_id": null,
      "original_filename": "原始文件名",
      "file_type": "txt",
      "file_size": 1024,
      "content": "文档内容",
      "status": "completed",
      "created_at": "2023-01-01T12:00:00",
      "updated_at": "2023-01-01T12:00:00",
      "processed_at": null
    }
  ]
  ```

#### POST /api/user/documents
- **功能**：创建新文档
- **请求体**：
  ```json

    "group_id": null,
    "original_filename": "原始文件名",
    "file_path": "文件路径",
  ```

- **响应**：
  ```json
  {
    "id": 1,
    "title": "文档标题",
    "original_filename": "原始文件名",
    "file_type": "txt",
    "file_size": 1024,
    "content": "文档内容",
    "status": "completed",
    "created_at": "2023-01-01T12:00:00",
    "updated_at": "2023-01-01T12:00:00",
  ```

#### PUT /api/user/documents/{document_id}
- **功能**：更新文档
- **路径参数**：

  ```json
  }
  ```

#### DELETE /api/user/documents/{document_id}
- **功能**：删除文档
- **路径参数**：
  - `document_id`：文档ID
- **响应**：
  ```json

  }
  ```

## 代码规范

### 1. 后端代码规范

#### Python代码规范
- 遵循PEP 8编码规范
- 使用4个空格缩进，禁止使用制表符
- 每行代码长度不超过100个字符
- 函数和方法使用小写字母加下划线的命名方式（snake_case）
- 类使用首字母大写的驼峰命名方式（CamelCase）
- 导入顺序：标准库、第三方库、本地模块
- 函数和方法必须添加文档字符串，使用Google风格
- 变量命名要清晰，避免使用单个字母（循环变量除外）

#### FastAPI规范
- 使用路由前缀（/api）
- 使用请求和响应模型（Pydantic）
- 错误处理使用HTTPException
- 日志记录使用Python内置的logging模块
- API文档使用Swagger UI（自动生成）

### 2. 前端代码规范

#### Vue代码规范
- 遵循Vue 3 Composition API
- 组件命名使用首字母大写的驼峰命名方式
- 使用单文件组件（.vue）
- 脚本部分使用setup语法糖
- 模板部分使用2个空格缩进
- CSS使用scoped属性，避免样式冲突
- 状态管理使用Pinia

#### JavaScript/TypeScript规范
- 变量和函数使用小写字母加下划线的命名方式（snake_case）
- 类使用首字母大写的驼峰命名方式（CamelCase）
- 每行代码长度不超过100个字符
- 使用const和let，禁止使用var
- 箭头函数优先于普通函数
- 模板字符串优先于字符串拼接

## 部署指南

### 1. 环境要求

#### 硬件要求
- CPU：至少4核心
- 内存：至少8GB
- 存储空间：至少50GB

#### 软件要求
- 操作系统：Windows 10/11或Linux（Ubuntu 20.04+）
- Python 3.12+ 
- Node.js 16+ 
- MySQL 8.0+ 

### 2. 本地部署

#### 后端部署
1. 进入backend目录
2. 安装依赖：`pip install -r requirements.txt --index-url https://pypi.tuna.tsinghua.edu.cn/simple`
3. 配置阿里云通义千问API Key：在`app/core/config.py`中设置`QWEN_API_KEY`
4. 启动后端服务：`uvicorn app.main:app --reload --host 0.0.0.0 --port 8000`

#### 前端部署
1. 进入frontend目录
2. 安装依赖：`npm install`
3. 启动前端服务：`npm run dev`
4. 在浏览器中访问：http://localhost:5173

### 3. 生产环境部署

#### 后端配置
- 使用Gunicorn作为WSGI服务器
- 配置Nginx作为反向代理
- 设置环境变量：
  - `ENVIRONMENT=production`
  - `DATABASE_URL=mysql+pymysql://user:password@host:port/database`
  - `SECRET_KEY=your_secret_key`
  - `QWEN_API_KEY=your_api_key`

#### 前端配置
- 构建生产版本：`npm run build`
- 将dist目录部署到Nginx或其他静态文件服务器
- 配置Nginx反向代理API请求

### 4. 模型配置

#### API Key配置
- 前往阿里云官网申请API Key：https://www.aliyun.com/product/dashscope
- 在`app/core/config.py`中配置`QWEN_API_KEY`
- 模型名称：`qwen-long`（长文本模型）

#### API调用优化
- 实现API调用缓存，减少重复请求
- 优化API调用参数（temperature=0.3, top_p=0.9, max_tokens=4096）
- 实现请求重试机制，提高API调用可靠性

## 开发流程

### 1. 分支管理
- `main`：主分支，用于发布生产版本
- `develop`：开发分支，用于集成所有功能分支
- `feature/xxx`：功能分支，用于开发新功能
- `bugfix/xxx`：bug修复分支，用于修复bug

### 2. 开发步骤
1. 从develop分支创建功能分支
2. 实现功能，编写测试
3. 提交代码，创建Pull Request
4. 代码审查
5. 合并到develop分支
6. 发布前合并到main分支

### 3. 测试流程
- 单元测试：使用pytest测试单个功能模块
- 集成测试：测试多个模块的协同工作
- 端到端测试：测试完整的业务流程
- 性能测试：测试系统的响应速度和资源占用

### 4. 代码审查
- 至少两名开发人员参与审查
- 检查代码规范和质量
- 检查功能实现是否正确
- 检查是否存在安全漏洞
- 检查测试覆盖率

## 安全规范

### 1. 数据安全
- 敏感数据加密存储（如密码使用bcrypt加密）
- 数据库连接使用环境变量配置
- 避免在日志中记录敏感信息

### 2. API安全
- 使用HTTPS协议
- 实现请求速率限制
- 验证请求参数
- 处理异常情况，避免泄露系统信息

### 3. 模型安全
- 保护API Key，避免泄露
- 监控API调用情况
- 定期更新SDK版本，修复安全漏洞

### 4. 部署安全
- 定期更新系统和依赖包
- 配置防火墙，限制访问端口
- 监控系统日志，及时发现异常情况
